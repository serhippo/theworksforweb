Class Diagram

Diagram yang menunjukkan hubungan antar kelas yang ada dalam sebuah perangkat lunak.

Apa itu Class?

Class adalah cetak biru (blueprint) untuk sebuah objek pada pemrograman berorientasi objek

Class digambarkan dalam bentuk tabel kecil dengan 3 lajur: 1 untuk nama Class, 1 untuk Attribute, 1 untuk Operasi

Berbeda dari video, kita akan menggunakan nama User yang mengacu pada User Instragram dari contoh Use Case Diagram.

Apa itu Attribute?

Attribute adalah data dengan nilai yang dapat mendeskripsikan sebuah Class.

Attribute diawali dengan huruf kecil, didahului oleh Visibility, dan diikuti dengan tipe data. Contohnya adalah - nama: string.

Tanda minus di awal attribut adalah Visibility bertipe Private. Pada umumnya Attribut pada program berorientasi objek menggunakan Private, sehingga attribut ini tidak bisa diakses dari luar kelas.

Hindari penggunaan attribut yang bisa didapatkan dari hasil perhitungan Attribute lain; contohnya Usia dan Total Harga Belanja, .

Apa itu Operation?

Operation adalah tindakan yang dapat dilakukan oleh Class di dalam perangkat lunak.

Operation diawali dengan dengan huruf kecil, didahului oleh Visibility, dan diikuti dengan Argument, yang biasanya dikosongkan. Contohnya adalah + lihatProfile().

Tanda minus di awal attribut adalah Visibility bertipe Private. Pada umumnya Operation pada program berorientasi objek menggunakan Public, sehingga Operation bisa diakses dari luar kelas.

Apa itu Inheritance / Generalization?

Mirip dengan Use Case, Interitance atau Generalization adalah hubungan antar Class menyatakan bahwa ada kesamaan Attribute dan Operation yang dapat digunakan oleh Class yang terkait

Generalization digambarkan dengan panah antar Class.

Class yang memanah akan memiliki Attribute yang sama dan dapat menjalankan semua Operation yang milik Class yang dipanah, tetapi tidak sebaliknya.

Sebagai contoh, Class Professional User akan memiliki Attribute nama dan Attribute lain milik User, tetapi User tidak dapat melakukan Operation lihatInsight yang dimiliki Professional User.

Apa itu Association dan Multiplicity?

Asociation menunjukkan hubungan antar Class, digambarkan dengan garis biasa.

Multiplicity adalah jumlah instansi Class yang berhubungan dengan Class lain, dituliskan dengan angka di atas garis Association.

Sebagai contoh, perhatikan angka antara User dan Post pada garis Association kedua Class ini. Di sisi User terdapat angka 1, sementara di sisi Post terdapat angka 0..*. Ini berarti 1 User bisa memiliki 0 atau banyak Post.

Apa itu Aggregation?

Aggregation adalah Association yang lebih spesifik, dimana keberadaan suatu Class tidak bergantung pada Class lain.

Aggregation digambarkan dengan wajik kosong pada ujung garis Association di sebuah Class.

Class di ujung wajik ini adalah Class induk, dan yang di ujung lainnya adalah Class anak. Jika Class induk ini dihapus, maka Class anak dapat dipindahkan ke Class lain.

Aggregation jarang digunakan dalam lingkungan Media Sosial, untuk bagian ini kita akan gunakan Steam dimanana setiap User, memiliki Game. Di saat User dihapus, maka Game dapat dipindahkan ke User lain.

Apa itu Composition

Composition adalah Association yang lebih spesifik, dimana keberadaan suatu Class bergantung pada Class lain.

Composition digambarkan dengan wajik penuh pada ujung garis Association di sebuah Class.

Class di ujung wajik ini adalah Class induk, dan yang di ujung lainnya adalah Class anak. Jika Class induk ini dihapus, maka Class anak akan ikut terhapus.

Sebagai contoh, Post di Instagram akan ikut terhapus jika User yang memiliki post ini dihapus.

Bagaimana contoh yang benar?

  1. Berapa banyak jumlah Post minimal yang dapat dimiliki oleh User?
  2. Apakah Professional Usermemiliki Attributecaption?
  3. Jika Post dihapus, apa yang terjadi dengan User?
  4. Jika Story dihapus, apa yang terjadi dengan Post?

Bagaimana contoh yang salah?

  1. Activity diubah menjadi Class. Activity diubah menjadi Operation, bukan Class.
  2. Attribute ganda di Class induk dan Class anak.
  3. Operation ganda di Class induk dan Class anak.
  4. Adanya Attribute Usia yang bisa didapatkan dari perhitungan.
