Activity Diagram

Diagram yang menunjukkan alur kegiatan pengguna dalam pemakaian sebuah perangkat lunak.

Apa itu Initial Node dan Final Node?

Initial Node adalah penanda awal satu set aktivitas dalam diagram ini.

Initial Node digambarkan dengan lingkaran penuh.

Final Node adalah penanda akhir satu set aktivitas dalam diagram ini.

Final Node digambarkan dengan lingkaran penuh dengan tambahan lingkaran luar.

Disarankan tiap diagram memiliki 1 Initial Node dan 1 Final Node.

Apa itu Activity dan Activity Edge?

Activity adalah kegiatan yang dilakukan di dalam perangkat lunak.

Activity digambarkan dalam bentuk persegi panjang bulat.

Nama Activity umumnya berstruktur Kata-Kerja-Kata-Benda.

Activity Edge adalah garis panah yang menunjukkan alur Activity.

Apa itu Fork Node dan Join Node?

Fork Node dan Join Node menyatakan beberapa alur aktivitas yang terjadi bersamaan.

Fork Node dan Join Node digambarkan dengan garis lurus tebal.

Fork Node memiliki satu Edge masuk dansetidaknya dua Egde keluar.

Join Node memiliki setidaknya dua Edge masuk dan satu Egde keluar.

Fork Node dan Join Node wajib berpasangan.

Apa itu Decision Node dan Merge Node?

Decision Node dan Merge Node menyatakan alur kegiatan yang berkondisi.

Decision Node dan Join Node digambarkan dengan wajik.

Kondisi kegiatan dituliskan pada Edge yang keluar dari Decision Node

Decision Node memiliki satu Edge masuk dan setidaknya dua Egde keluar.

Merge Node memiliki setidaknya dua Edge masuk dan satu Egde keluar.

Pada umumnya Decision Node dan Merge Node wajib berpasangan.

Apa itu Accept Event Action?

Accept Event Action atau Trigger menyatakan kegiatan yang dapat terjadi kapanpun jika suatu syarat tercapai.

Accept Event Action digambarkan bendera.

Accept Event Action bersatu dengan alur kegiatan utama melalui sebuah Merge Node yang tidak memiliki Decision Node.

Bagaimana contoh yang benar?

  1. Diagram boleh disusun vertikal, horizontal, atau gabungan keduanya.
  2. Apakah kegiatan yang dilakukan setelah user melakukan Logout?
  3. Apakah Verifikasi Password dan Verifikasi Username adalah pilihan?
  4. Mengapa Merge Node [Verifikasi gagal] tidak berada setelah Decision Node-nya?

Bagaimana contoh yang salah?

  1. Ada lebih dari satu Activity Edge yang masuk ke sebuah Activity.
  2. Hanya ada 1 kegiatan di aantara Fork Node dan Join Node.
  3. Adanya Decision Node tanpa Merge Node.
  4. Adanya Decision Node tanpa syarat kondisi.
  5. Accept Event Action tidak menyatu ke diagram utama.
  6. Activity Edge tidak berupa garis panah.
  7. Ada lebih dari satu Activity Edge yang keluar dari sebuah Activity.
  8. Activity tidak berkelanjutan atau buntu.
  9. Ada dua Activity Edge menuju Final Node. Tidak sepenuhnya salah, tapi sebaiknya dihindari.
