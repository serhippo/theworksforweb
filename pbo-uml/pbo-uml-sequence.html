Sequence Diagram

Diagram interaksi yang menggambarkan pengiriman pesan antar lifeline.

Apa itu Lifeline?

Lifeline adalah instansi yang berpartisipasi dalam interaksi yang akan digambarkan oleh diagram.

Lifeline digambarkan dalam bentuk persegi panjang. Ada garis putus-putus yang membentang tegak lurus dengan persegi panjang untuk melambangkan waktu.

Bentuk persegi panjang ini dapat dilambangkan dengan orang-orangan untuk Actor, tapi tidak wajib.

Ada persegi panjang di atas garis putus-putus ini yang melambangkan seberapa lama instansi ini aktif. Persegi panjang ini tidak dibutuhkan untuk Actor.

Kita akan menggunakan contoh login User dari Instragram berdasarkan Use Case Diagram.

Apa itu Message Line?

Message Line adalah perintah yang dikirimkan dalam bentuk pesan dari satu Lifeline ke Lifeline lain.

Message Line digambarkan dengan garis panah dengan label Operation dari Class Diagram.

Tidak ada Message Line yang segaris! Tiap Message Line baru selalu dimulai lebih turun dari sebelumnya! Perhatikan pada contoh: posisi verifikasiLogin() yang terjadi belakangan berada lebih rendah dari login().

Apa itu Return Line?

Return Line adalah respon dari perintah yang diterima melalui Message Line.

Return Line digambarkan dengan garis panah putus-putus menuju instansi yang melakukan Message Line.

Return Line umumnya tidak memiliki label, tetapi bisa diberi label unuk memperjelas.

Return Line dan Message Line wajib satu pasang!

Tidak ada Return Line yang segaris! Tiap Return Line baru selalu dimulai lebih turun dari sebelumnya!

Apa itu Alternative Frame?

Alternative Frame adalah penanda bagian pada diagram yang memiliki lebih dari satu kemungkinan kejadian.

Alternative Frame digunakan di saat yang mirip dengan Decision Node pada Activity Diagram.

Alternative Frame digambarkan berupa persegi yang melingkupi kemungkikan kejadian. Syarat alternatif dituliskan, seperti [login berhasil], sementara alternatif tindalan ditulis [else]

Bagaimana contoh yang benar?

  1. Pastikan diagram yang dibuat sesuai dengan diagram sebelumnya!

Bagaimana contoh yang salah?

  1. Ada message line yang segaris, yaitu login() dan verifikasiLogin().
  2. Alternative Frame tanpa tulisan Alternative.
  3. Alternative Frame tanpa tulisan kondisi dan Else.
  4. Message Line tanpa Return Line.
  5. Lifeline non-Actor tanpa persegi panjang yang melambangkan keaktifan.

Tugas

Buatlah Sequence Diagram berdasarkan topik dari dosen!

Penambahan Nilai berdasarkan (maksimal 100):

  • Ada 2 lifeline yang benar: +15 point
  • Ada 4 lifeline yang benar: +10 point
  • Ada 8 lifeline yang benar: +5 point
  • Ada 16 lifeline yang benar: +5 point
  • Ada 1 Alternative frame yang benar: +15 point
  • Ada 2 Alternative frame yang benar: +10 point
  • Ada 4 Alternative frame yang benar: +5 point
  • Ada 8 Alternative frame yang benar: +5 point
  • Ada 4 pasang Message Line dan Return Line yang benar: +15 Point
  • Ada 8 pasang Message Line dan Return Line yang benar: +10 Point
  • Ada 16 pasang Message Line dan Return Line yang benar: +5 Point
  • Ada 32 pasang Message Line dan Return Line yang benar: +5 Point
  • Ada 64 pasang Message Line dan Return Line yang benar: +5 Point

Pengurangan Nilai berdasarkan:

  • Ada message line yang segaris.
  • Alternative Frame tanpa tulisan Alternative.
  • Alternative Frame tanpa tulisan kondisi dan Else.
  • Message Line tanpa Return Line.
  • Lifeline non-Actor tanpa persegi panjang yang melambangkan keaktifan.

Catatan:

  • Penambahan Nilai maksimal adalah 100.
  • Pengurangan Nilai pertama bernilai -30, selebihnya -5.
  • Nilai akhir adalah: Penambahan Nilai - Pengurangan Nilai
  • Mahasiswa diizinkan merevisi tugas selama batas waktu pengumpulan dan revisi belum dilewati.
