Use Case Diagram

Diagram yang menunjukkan "apa yang bisa dilakukan perangkat lunak".

Apa itu System?

System adalah perangkat lunak yang akan Anda buat, bisa berupa aplikasi, game, bahkan website.

System digambarkan dalam bentuk persegi panjang untuk meletakkan use case anda nantinya.

Nama System diletakkan di bagian atas persegi panjang,umumnya dinyatakan dalam nama perangkat lunak yang Anda buat.

System menggambarkan ruang lingkup perangkat lunak Anda. Use Case yang anda letakkan di dalam System adalah Use Case yang terjadi langsung dalam perangkat lunak tersebut.

Apa itu Actor?

Actor adalah apapun yang berinteraksi dengan System.

Actor umumnya adalah manusia, tapi bisa juga organisasi, atau program lain.

Actor digambarkan dalam bentuk orang-orangan.

Actor terbagi atas Primary Actor dan Secondary Actor.

Primary Actor menginisiasi penggunaan System dan ditempatkan di sebelah kiri System.

Secondary Actor melakukan reaksi terhadap inisiasi Primary Actor dan ditempatkan di sebelah kanan System.

Pada umumnya perangkat lunak hanya memiliki Primary Actor saja.

Apa itu Use Case?

Use Case melambangkan kegiatan yang dilakukan sistem atas inisiasi (Primary) Actor.

Use Case digambarkan dalam bentuk oval, dan ditempatkan dalam System.

Hubungan antara Primary Actor penginisiasi dan Use-Case-nya digambarkan dengan garis biasa.

Nama Use Case umumnya berstruktur Kata-Kerja-Kata-Benda.

Satu Use Case umumnya hanya diinisiasi oleh satu Primary Actor.

Use Case yang berhubungan dengan lebih dari satu Primary Actor bearti Use Case tersebut memerlukan lebih dari satu Primary Actor untuk dapat berjalan (sangat tidak lazim).

Apa itu include?

Include menyatakan hubungan antara 2 Use Case, dimana Use Case yang dinisisiasi oleh Actor (Base Use Case) akan ikut menjalankan Use Case yang dipanah (Included Use Case).

Include digambarkan dalam bentuk panah terputus dengan tulisan <<include>> di tengahnya.

Included Use Case atau Use Case yang dipanah, tidak perlu berhubungan dengan Actor karena akan berjalan begitu Actor menjalankan Base Use Case-nya.

Sebagai contoh: Login --<<include>>--> Verifikasi Password. Ini berarti di saat Actor melakukan Login, maka Verifikasi Password akan berjalan setidaknya satu kali selama proses Login berjalan.

Apa itu extend?

Extend juga menyatakan hubungan antara 2 Use Case, dimana ketika Use Case yang dinisisiasi oleh Actor (Base Use Case) memenuhi syarat tertentu, maka Use Case yang memanah (Extend Use Case) baru akan berjalan.

Extend digambarkan dalam bentuk panah terputus dengan tulisan <<extend>> di tengahnya.

Mirip dengan Include, Extend Use Case atau Use Case yang memanah, tidak perlu berhubungan dengan Actor karena akan berjalan begitu Actor menjalankan Base Use Case-nya.

Sebagai contoh: Login <--<<extend>>-- Tampilkan Pesan Eror Login. Ini berarti di saat Actor melakukan Login, maka Tampilkan Pesan Eror Login hanya akan berjalan jika kondisi tertentu terpenuhi, dalam kasus ini, di saat Actor salah memasukkan informasi Login.

Apa itu Generalization?

Generalization adalah hubungan antar Actor yang menyatakan bahwa ada kesamaan Use Case yang dapat digunakan oleh Actor-Actor terkait.

Generalization digambarkan dengan panah antar Actor.

Actor yang memanah akan dapat menjalankan semua Use Case yang berhubungan dengan Actor yang dipanah, tetapi tidak sebaliknya.

Sebagai contoh, Actor Professional User dapat melakukan Login dan Use Case lain milik User, tetapi User tidak dapat melakukan Lihat Insight yang hanya berasosiasi dengan Professional User.

Bagaimana contoh yang benar?

  1. Apakah nama perangkat lunak yang dibuat?
  2. Ada berapa include dan berapa extend?
  3. Apa sajakah Use Case yang dapat diinisiasi oleh Professional User?
  4. Apakah Menggunakan Filter akan berjalan saat User melakukan Buat Post?

Bagaimana contoh yang salah?

  1. Use Case Reel hanya memiliki kata benda, sehingga kegiatan Actor-nya kurang jelas
  2. Generalisasi seperti ini berarti Userdapat melakukan Lihat Insight, tetapi Professional User hanya dapat melakukan Lihat Insight saja dan tidak bisa melakukan Login
  3. Use Case yang bukan merupakan kegiatan, seperti Tidak mengubah profil.
  4. Use Case di luar perangkat lunak, seperti Menerima OTP.
  5. Penggunaan <<include>> atau <<extend>> yang kurang tepat: Di saat User melakukan Buat Post, maka User akan langsung dipaksa menggunakan Mengunggah gambar, Mengunggah video, dan Menggunakan filter.
  6. Garis hubungan antar Actor dan Use Case menggunakan panah. Garis ini hanya boleh berupa garis biasa.
  7. Adanya garis biasa yang mengubungkan Use Case. Hubungan antar Use Case hanya boleh berupa <<include>> atau <<extend>>
